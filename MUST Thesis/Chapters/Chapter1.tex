\section{Android архитектурын судалгаа}
Орчин үеийн Android хөгжүүлэлтэд MVVM (Model-View-ViewModel) архитектур нь албан ёсны санал болгож буй стандарт юм. Энэхүү архитектур нь UI логикийг өгөгдлийн логикоос салгаж, аппликейшныг тестлэх болон хөгжүүлэхэд хялбар болгодог.
\begin{itemize}
    \item \textbf{Model}: Room өгөгдлийн сангийн entities (\texttt{Conversation}, \texttt{Category}) болон Repository давхарга.
    \item \textbf{View}: Activity болон Fragment-ууд. ViewBinding technologies ашиглан XML-тэй холбогдоно.
    \item \textbf{ViewModel}: UI-д зориулсан өгөгдлийг бэлтгэж, \texttt{StateFlow} эсвэл \texttt{LiveData} ашиглан өгөгдлийг дамжуулна. Энэ нь Configuration Change (жишээ нь, дэлгэц эргүүлэх эсвэл нугалах) үед UI state-ийг хадгалж үлддэг.
\end{itemize}

\section{Kotlin, Coroutines болон Flow}
Асинхрон ажиллагаа болон өгөгдлийн урсгалыг удирдах нь мессежийн апп-ын хувьд маш чухал. 10,000+ мессеж ачаалахад UI thread гацах ёсгүй. Үүнийг Kotlin Coroutines ашиглан background thread-д гүйцэтгэнэ. \texttt{Flow} технологийг ашиглан өгөгдлийн санд өөрчлөлт орох бүрт UI автоматаар шинэчлэгдэх ``Reactive Programming'' зарчмыг хэрэгжүүлсэн~\cite{kotlin_programming}.

\section{Room өгөгдлийн сан}
Room нь SQLite дээр суурилсан өгөгдлийн сангийн удирдлагын систем (ORM) юм. Төслийн явцад өгөгдлийн сан v10-аас v13 руу шилжсэн. v13 хувилбарт бид дараах өөрчлөлтүүдийг хийсэн:
\begin{itemize}
    \item \texttt{plain\_keywords}: Түлхүүр үгс хадгалах талбар.
    \item \texttt{regex\_patterns}: Регуляр илэрхийлэл хадгалах талбар.
    \item \texttt{thread\_color}: Харилцан яриа бүрт оноох тусгай өнгө.
\end{itemize}
Шилжилт (Migration) хийхдээ \texttt{MigrationTestHelper} ашиглан data loss байхгүй гэдгийг баталгаажуулсан~\cite{room_guide}.

\section{Foldable төхөөрөмжийн дэмжлэг}
Нугалдаг төхөөрөмжүүд нь дэлгэцийн хэмжээ нь динамикаар өөрчлөгддөг онцлогтой. Үүнийг \texttt{WindowManager} сангийн \texttt{WindowInfoTracker} ашиглан мэдэрч, UI-г хоёр баганатай (master-detail) харагдац руу шилжүүлнэ. Дэлгэц нугалагдах үед \texttt{savedInstanceState} болон \texttt{ViewModel}-ийг ашиглан хэрэглэгчийн scroll position-ийг хадгалж үлддэг.
