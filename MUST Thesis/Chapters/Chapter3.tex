\chapter{Шаардлага ба төлөвлөлт}

\section{Функционал шаардлага (Functional Requirements)}
Системийн гүйцэтгэх үндсэн үйлдлүүдийг SRS стандартын дагуу дараах байдлаар тодорхойлов.
\begin{enumerate}[label=FR\arabic*.]
    \item \textbf{Regex Pattern Matching}: Хэрэглэгчийн тохируулсан тогтмол илэрхийлэл ашиглан ирж буй зурвасыг автоматаар таньж, ангилах.
    \item \textbf{Динамик хавтас (Folder) удирдлага}: Хэрэглэгч өөрийн хэрэгцээнд тааруулан шинэ хавтас үүсгэх, нэр өнгийг засах, устгах.
    \item \textbf{Харилцан ярианы шошго (Category Labeling)}: Харилцан яриа бүрт тухайн зурваст тохирох өнгөт шошго оноож, харагдах байдлыг ялгах.
    \item \textbf{Adaptive UI Support}: Foldable төхөөрөмж нээгдэх үед интерфейсийг хоёр баганат харагдац руу гацалтгүйгээр шилжүүлэх.
    \item \textbf{Шудрах (Swipe) үйлдлийн тохиргоо}: Харилцан яриаг баруун, зүүн тийш шудрах үйлдлээр устгах, архивлах зэрэг үйлдлийг хурдан гүйцэтгэх.
    \item \textbf{Хугацааны бүлэглэлт}: Inbox дахь зурвасуудыг Өнөөдөр'', Өчигдөр'', Энэ сар'' гэсэн гарчиг дор автоматаар бүлэглэх.
\end{enumerate}

\section{Функционал бус шаардлага (Non-Functional Requirements)}
Системийн чанар болон найдвартай ажиллагааг хангах дараах шаардлагуудыг тавьсан:
\begin{itemize}
    \item \textbf{Гүйцэтгэл}: 1,000-аас дээш харилцан яриатай үед жагсаалт секундэд 60 frame хурдтай (smooth scroll) ажиллах.
    \item \textbf{Аюулгүй байдал}: Зурвасын агуулгыг гадагш дамжуулахгүй, бүх шүүлтүүрийг локал төхөөрөмж дээр (On-device) ажиллуулах.
    \item \textbf{Найдвартай байдал}: Room өгөгдлийн сангийн шилжилт (Migration) хийх үед хэрэглэгчийн өмнөх өгөгдөл алдагдахгүй байх.
    \item \textbf{Уян хатан байдал}: Шүүлтүүрийн дүрмийг хэрэглэгч хүссэн үедээ өөрчлөхөд систем бодит хугацаанд хариу үзүүлэх.
\end{itemize}

\section{Ажлын хүрээ (Scope Statement)}
Төслийн ажлын хүрээг MoSCoW аргачлалаар эрэмбэлж тодорхойлов.
\begin{itemize}
    \item \textbf{Must-Have}: Regex Engine, Room v13 Migration, Динамик хавтаслах систем.
    \item \textbf{Should-Have}: Foldable Layout Continuity, Swipe Actions, Sticky Headers.
    \item \textbf{Could-Have}: Зурвасын агуулгад тулгуурласан ухаалаг мэдэгдэл (Smart Notifications).
    \item \textbf{Won't-Have}: Cloud Backup, RCS (Rich Communication Services) дэмжлэг.
\end{itemize}

\section{Хөгжүүлэлтийн төлөвлөгөө (Development Plan)}
Хөгжүүлэлтийн ажлыг Iterative (Давталттай) аргачлалаар гүйцэтгэсэн бөгөөд доорх үе шатуудад хуваав:
\begin{enumerate}
    \item \textbf{I үе шат}: Суурь өгөгдлийн сангийн өргөтгөл (v10 $\rightarrow$ v13) болон Regex шүүлтүүрийн алгоритм хөгжүүлэлт.
    \item \textbf{II үе шат}: Хавтас удирдлагын систем болон Swipe үйлдлүүдийн UI хөгжүүлэлт.
    \item \textbf{III үе шат}: Foldable төхөөрөмжийн дасан зохицох харагдац болон Continuity логик.
    \item \textbf{IV үе шат}: Гүйцэтгэлийн оновчлол, тест ба чанарын баталгаажуулалт.
\end{enumerate}
Хөгжүүлэлтийн явцад Git-ийн Feature Branch'' стратегийг ашиглаж, долоо хоног бүр явцын түүхийг баталгаажуулсан.
