\section{Системийн архитектур}
Төслийн архитектурыг дараах гол хэсгүүдэд хувааж болно:
\begin{itemize}
    \item \textbf{Filtering Engine}: Ирж буй зурвасыг ``Intercept'' хийж, \texttt{Category} дүрмүүдийн дагуу \texttt{folder\_id} оноох логик.
    \item \textbf{UI Layouts}: RecyclerView-ийн Sticky Headers (Age Grouping) болон Foldable хоёр баганат харагдац.
    \item \textbf{Data Management}: Room v13 сангийн өргөтгөл болон SharedPreferences (JSON Mapping) ашигласан хавтаслах систем.
\end{itemize}

\textbf{Диаграммын бүрэлдэхүүн хэсгүүдийн тайлбар:}
\begin{itemize}
    \item \textbf{Data Layer (Өгөгдлийн давхарга)}: Room DB (v13) болон DAO (Data Access Objects) хэсгээс бүрдэх бөгөөд зурвас, ангилал, хавтасны мэдээллийг SQLite форматаар локал төхөөрөмж дээр хадгална.
    \item \textbf{Repository / Context Extensions}: Төслийн бизнес логикийн цөм хэсэг. Ирж буй зурвасыг Regex-ээр шалгаж ангилах (Filtering Engine) болон өгөгдлийн урсгалыг (Flow) удирдах үйлдлүүд энд ажиллана.
    \item \textbf{Presentation Layer (Танилцуулга давхарга)}: ViewModel, Activity болон Compose UI хэсгүүдийг агуулна. ViewModel нь өгөгдлийг UI-д харуулахад бэлэн ``State'' (төлөв) болгон хувиргаж, дэлгэц нугалах үед (Continuity) өгөгдлийг алдалгүй хадгалах үүрэгтэй.
\end{itemize}

\section{Өгөгдлийн загварчлал}
Room v13 схемд \texttt{plain\_keywords} болон \texttt{regex\_patterns} талбаруудыг нэмж оруулсан. Энэ нь шүүлтүүрийг илүү нарийвчлалтай болгосон.
\begin{lstlisting}[language=Kotlin, caption=Category Entity-ийн бүтэц]
@Entity(tableName = "categories")
data class Category(
    @PrimaryKey(autoGenerate = true) var id: Long = 0,
    var name: String,
    var color: Int,
    var plainKeywords: String = "",
    var regexPatterns: String = ""
)
\end{lstlisting}
